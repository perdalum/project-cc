# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**Project Command and Control** is a native macOS app for personal project/case management. The goal is to give the user deliberate command and control over all their projects and areas of responsibility.

Key documents:
- `SPEC.md` — full data model, UI spec, and feature requirements
- `THEORY.md` — conceptual background
- `AGENT.md` — agent instructions (role, language, philosophy)

## Development Stack

- **Language:** Swift
- **UI Framework:** SwiftUI (best practices, native macOS widgets)
- **Persistence:** JSON file at a configurable folder path; read on launch, written immediately on every update

## Coding Philosophy

From `AGENT.md`: write the simplest, most direct code possible. No preemptive optimization or generalization. Run every significant decision by the user, but treat the user as an expert and push back on requirements when warranted. Use TDD: write tests before implementation code.

## Data Model

A **Project** has these properties (UW = user-writable, UR = user-readable):

| Property | Type | Notes |
|---|---|---|
| Name | String | UW |
| Category | String | UW — e.g. CHC, Arts, AU, Ego, Family |
| Project type | Enum | UW — Classical project or Area of responsibility |
| State | Enum | UW — Idea, New, Active, Delegated, Waiting, Rejected, Done |
| Log | [{date, oldState, newState, comment}] | UR — auto-populated on state change; comment mandatory for Delegated/Waiting/Rejected |
| Start | Date | UW |
| End | Date | UW |
| Modified | Date | UR |
| Folder | macOS path | UW — opens in Finder |
| Notes | [{date, text}] | UW |
| Touched | [Date] | UW — append "now" on click |
| Latest Review | Date | UW |
| Next | Date | UW |
| URL | URL | UW |
| Goal | Text | UW — editable in a separate window |

## UI Structure

Three main screens:

1. **Project List View** — table of all projects with sortable columns; filter bar at top (text search on Name; supports `-` for NOT, `|` for OR, implicit AND); filter by State; calculated field: "Terminal" (opens Folder in Terminal.app)
2. **Project Property View** — all properties for a single project, editable via native macOS widgets
3. **Overview View** — calendar heat map showing updates/changes/touches per project; same filter options as list view

## Build & Test

```bash
./build.sh           # build Release .app → dist/ProjectCommandAndControl.app (and opens it)
./build.sh Debug     # build Debug variant
swift test           # run all tests (exit 0 = all pass; Swift Testing only prints on failure)
```

**Why `build.sh` instead of `xcodebuild` directly:** the project tree is on iCloud Drive, which adds xattr metadata that codesign rejects. `build.sh` builds to `/tmp` then copies the signed `.app` to `dist/`.

**Xcode project** is generated by `xcodegen` from `project.yml`. Regenerate after adding files:
```bash
xcodegen generate
```

Tests use **Swift Testing** (`import Testing`, `#expect`). The `Package.swift` carries `unsafeFlags` on the test target to locate `Testing.framework` under CommandLineTools — keep them.

## Running in Xcode

Open `ProjectCommandAndControl.xcodeproj`. Minimum deployment target: **macOS 26**.
